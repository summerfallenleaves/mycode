### 续：IME 候选框定位修复

#### 问题
- macOS 拼音输入法候选框挤占界面第一行，而非显示在输入框附近
- 原因：blessed 使用虚拟光标（`{inverse}` 标签模拟），终端真实光标位置不受控；IME 通过 `firstRectForCharacterRange` 读取终端光标坐标定位候选框

#### 方案
- **方案 A2**：每次 `screen.render()` 后用 `program.cup()` 将真实光标移到输入框光标位置，不显示光标
- 利用 blessed 内置的 `blessed.unicode.strWidth()` 计算 CJK 双宽字符的实际列宽
- 非输入模式（model select、resume list、connect wizard）跳过光标定位

#### 修改
- **`packages/cli/src/app.ts`** — `update()` 函数末尾添加 `positionCursorForIme()` 调用：
  - `strWidth` = `blessed.unicode.strWidth` 计算输入文本的视觉宽度
  - `row` = `screen.height - 1`（inputBox 在 bottom:0）
  - `col` = `2 + strWidth(input.slice(0, cursorIndex))`（"> " 前缀 + 光标前文本宽度）
  - `screen.program.cup(row, col)` 移动光标

#### 验证
- `pnpm --filter @my-agent/cli build` ✅（含 DTS）

### 续：AI SDK 空响应错误健壮性增强

#### 问题
- 运行中偶发 `AI_NoOutputGeneratedError: No output generated` 导致进程异常终止
- 原因：DeepSeek API 偶尔返回空响应，AI SDK 在流结束时检测到无输出即抛出此错误
- 进程崩溃根因：`result.response`（`PromiseLike`）的 rejection 未被捕获，变成 unhandled rejection 导致 Node.js 进程退出

#### 修改
- **`packages/core/src/llm/adapter.ts`** — 三处增强：
  1. `responsePromise` 用 `Promise.resolve()` 包裹后加 `.catch(() => [])`，防止 unhandled rejection
  2. `for await` 循环外层加 `try/catch`，识别 `AI_NoOutputGeneratedError`（`hasOutput === false`）时 yield 降级提示文本
  3. 非 `AI_NoOutputGeneratedError` 的其他流错误照常抛出，由 agent.ts 的 catch 处理
- **`packages/core/src/agent.ts`** — 无需修改：已有的 `catch` 块会 yield `error` 事件并优雅结束会话，前端显示错误信息供用户重发

#### 设计决策
- 不在 agent 层加自动重试：错误后 `this.messages` 已包含当前用户消息，自动重试会导致消息重复
- 用户在前端重新发送即可触发新一轮对话
